# Демонстрационный экзамен 2026 — Вариант 1 «ЧитайГород»

Шесть **независимых** решений одного и того же задания (магазин книг ООО «ЧитайГород»).
Технология — **Flask** (веб-приложение). Каждое решение лежит в своей папке, полностью
самодостаточно (свой код, своя БД/скрипт сидинга, свои ресурсы, свои PDF-диаграммы) и реализует
**все 4 модуля** задания. Архитектуры, имена и подходы у решений — РАЗНЫЕ (см. таблицу).

> Все 6 решений проверены в реальном браузере (Playwright + Chromium): вход, гость, роли,
> подсветка скидок, поиск/сортировка/фильтр, формы товаров и заказов, обработка ошибок.
> Скриншоты корректной работы — в `docs/screenshots/` каждой папки.

## Как запустить (нужен Python 3.11+, пакеты уже могут быть установлены глобально)

В каждой папке решения сначала (по желанию) `pip install -r requirements.txt`, затем:

| Папка | Заполнить БД | Запустить сервер |
|-------|--------------|------------------|
| `01_classic_orm_blueprints` | `python seed.py` | `python run.py` |
| `02_sqlite3_monolith` | (создаётся при первом запуске) или `python -c "import db; db.init_db(force=True)"` | `python app.py` |
| `03_layered_jwt` | `python manage.py init-db` | `python manage.py run` (или `python wsgi.py`) |
| `04_cli_dotenv_mysql` | `python manage.py initdb && python manage.py importdata` | `python manage.py run` |
| `05_sqlalchemy_core` | `python seed_cli.py` | `python run.py` |
| `06_functional_minimal` | `python seed.py` | `python app.py` |

Сервер поднимается на `http://127.0.0.1:5000`. Подробности — в `README.md` каждой папки.

## Соответствие модулям задания

- **Модуль 1 — БД средствами СУБД.** Схема в 3НФ (11 таблиц, PK/FK, ссылочная целостность,
  `PRAGMA foreign_keys=ON`), `schema.sql` (DDL), сидинг из очищенных данных, `docs/er_diagram.pdf`.
- **Модуль 2 — Алгоритм и приложение.** Экран входа (Argon2), вход гостем, роли (гость/клиент/
  менеджер/админ), logout, ФИО в правом верхнем углу, логотип + favicon, список товаров по макету,
  подсветка: скидка > 25 % → фон `#23E1EF`; снижена цена → старая цена перечёркнута красным +
  итоговая чёрным; нет на складе → серый (приоритет над голубым). Руководство по стилю (Comic Sans MS,
  `#FFFFFF`/`#ABCFCE`/`#546F94`).
- **Модуль 3 — Последовательный UI + CRUD товаров.** Навигация и «Назад», заголовки, типизированные
  сообщения (ошибка/предупреждение/информация), поиск (real-time, по всем текстовым полям),
  сортировка (цена/количество, ↑↓), фильтр по скидке («Все диапазоны» / 0–12,99 / 13–16,99 / 17+),
  совместная работа поиск+фильтр с сохранением сортировки; форма add/edit (выпадающие списки
  категории и поставщика), валидация (цена ≥ 0 с сотыми, количество ≥ 0), ID авто `max+1` /
  read-only при edit, загрузка фото с ресайзом до **300×200** и удалением старого, запрет двух окон
  редактирования, запрет удаления товара, входящего в заказ; `docs/flowchart.pdf` (ГОСТ 19.701-90).
- **Модуль 4 — Заказы.** Кнопка «Заказы» (менеджер/админ), список по макету (состав-артикул, статус,
  адрес пункта выдачи, дата заказа, дата доставки), CRUD заказов (только админ; менеджер — просмотр),
  форма (состав-артикул, статус и пункт выдачи — выпадающие, даты), обновление списка.
